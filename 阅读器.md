我要做一个类似 Legado/阅读 的简化版项目，用来理解它的核心原理。

目标：做一个“固定网站解析版小说阅读器 Demo”，先只支持一个固定小说网站，不做用户自定义书源。

核心流程：
搜索关键词 → 请求网站搜索页 → 解析搜索结果 → 点击书籍 → 解析章节目录 → 点击章节 → 解析正文 → 阅读展示。

技术栈建议：
前端：React + TypeScript + Vite
后端：Node.js + Express + Axios + Cheerio

项目页面：
1. 搜索页：输入关键词，展示搜索结果
2. 章节目录页：展示一本书的章节列表
3. 阅读页：展示章节标题和正文

后端核心模块：
bookParser.ts

需要实现三个方法：
1. searchBooks(keyword)
   - 拼接搜索 URL
   - 请求 HTML
   - 用 CSS Selector 解析书名、作者、详情页链接
   - 返回 Book[]

2. getChapters(bookUrl)
   - 请求书籍详情页 HTML
   - 用 CSS Selector 解析章节名和章节链接
   - 返回 Chapter[]

3. getChapterContent(chapterUrl)
   - 请求章节页 HTML
   - 用 CSS Selector 解析标题和正文
   - 清洗正文
   - 返回 ChapterContent

数据模型：
Book = {
  title: string;
  author?: string;
  url: string;
  cover?: string;
  description?: string;
}

Chapter = {
  title: string;
  url: string;
}

ChapterContent = {
  title: string;
  content: string;
}

后端 API：
GET /api/search?keyword=xxx
GET /api/chapters?url=xxx
GET /api/content?url=xxx

基础工具函数：
1. normalizeUrl：把相对链接转成绝对链接
2. cleanContent：清理正文空格、换行、广告文本
3. httpClient：统一设置 User-Agent 请求网页

第一版不要做：
- 用户自定义书源
- 多书源搜索
- 登录 Cookie
- JS 执行
- 听书
- EPUB/TXT
- 书架同步
- 复杂翻页动画
- 账号系统

第一版验收标准：
1. 能搜索书籍
2. 能显示搜索结果
3. 能点击书籍进入章节目录
4. 能点击章节进入阅读页
5. 能显示章节正文
6. 能处理请求失败、解析失败、空结果
7. 代码结构清晰，解析逻辑集中在 bookParser.ts

项目目的：
先理解 Legado 的核心机制：请求网页 → 解析 HTML → 提取书籍/章节/正文 → 转成统一数据 → 展示阅读界面。

固定网站解析版就是把规则写死在代码里；Legado 是把这些规则抽象成用户可配置的书源系统。
